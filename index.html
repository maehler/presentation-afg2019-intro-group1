GWAS and potential P-hacking

A functional genomics project

Background

    Studies where teams perform association studies between a trait and a subset of SNPs within genes that are assumed to contribute to the trait variation.

P-hacking

The (possibly unintentional) misuse of statistical methods in order to obtain significant P-values.

Your task

Run regular GWAS as well as GWAS on a subset of SNPs that are located in genes differentially expressed for the trait.

Data

  • Gene expression: ~35,000 genes, 80 genotypes (htseq-count)
  • Genetic variation: ~4 million SNPs (VCF format)
  • Phenotype data: leaf shape (BLUPs)
    • Area
    • Indent depth
    • Circularity
  • Gene annotations (gff3 format)

Choices

  • Do DE yourselves or get the genes
  • Perform the regular GWAS or get our old results

Thoughts

  • Do you find more significant assocations?
  • Do these in that case make biological sense?
  • Is this cheating?

Problems you might run into

  • Extracting SNPs within a set of genes
  • Converting SNP data to formats that can be used in GWAS
  • Differential expression for a continuous trait

Software

  • bcftools (working with VCF files; command line)
  • bedtools (genome arithmetic; command line)
  • plink (file formats SNPs; command line)
  • GEMMA (GWAS; command line)
  • DESeq2 (differential expression; R-package)
  • Downstream analysis in R or Python (or any other language you like)
